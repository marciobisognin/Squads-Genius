# Complaint to Skill

## Quando usar
Use quando uma reclamação ou tarefa repetida puder virar skill reutilizável.

## Procedimento
1. Identificar dor
2. Confirmar recorrência
3. Definir entradas e saídas
4. Escrever SKILL.md
5. Testar com exemplo

## Saída esperada
- Artefato Markdown claro.
- Critérios de qualidade.
- Próximo passo recomendado.
- Indicação de necessidade de aprovação humana quando houver ação externa.
