# R&D — Fundamentos do Proactive AI OS Squad

## Tese de pesquisa
O ganho de produtividade com IA cresce quando o usuário deixa de tratar o modelo como chat reativo e passa a construir um sistema com memória, contexto, skills e automações proativas.

## Princípios extraídos do vídeo analisado
1. Reclamações são insumos de design de sistemas.
2. A IA deve entrevistar o usuário para reduzir ambiguidade.
3. Tarefas repetidas devem virar skills.
4. Rotinas previsíveis devem ser agendadas.
5. Contexto deve ser organizado em arquivos portáveis.
6. Agentes devem operar em níveis: microtarefa, companheiro, delegado e colega de equipe.
7. A autonomia humana deve ser preservada.
8. O melhor sistema é incremental: uma dor, uma skill, uma rotina, uma expansão.

## Hipóteses
- Context packs reduzem retrabalho e aumentam personalização.
- Skills reduzem custo cognitivo em tarefas recorrentes.
- Rotinas proativas geram maior valor que prompts manuais repetidos.
- Revisão crítica reduz risco de confiança excessiva em IA.

## Experimentos v1
1. Transformar uma reclamação em proposta de skill.
2. Gerar briefing matinal sintético a partir de arquivos de contexto.
3. Criar context pack mínimo do usuário.
4. Validar fluxo com checklist de autonomia.
